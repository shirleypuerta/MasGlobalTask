</a>framework. Appium allows native, hybrid and web application testing and supports automation test on physical devices as well as an emulator or simulator both. It offers cross-platform application testing, i.e.&nbsp;single API works for both Android and iOS platform test scripts. </p><p>It has <strong>NO</strong> dependency on Mobile device OS. Because APPIUM has framework or wrapper that translate<a href="/selenium-tutorial.html"> Selenium </a>Webdriver commands into UIAutomation (iOS) or UIAutomator (Android) commands depending on the device type, not any OS type. </p><p>Appium supports all languages that have Selenium client libraries like- Java, Objective-C,<a href="/interactive-javascript-tutorials.html"> JavaScript </a>with node.js, PHP, Ruby, Python,<a href="/c-tutorial.html"> C#, </a>etc. </p><p>In this tutorial, we will learn about </p><ul> <li><a href="#1">How APPIUM Works?</a></li> <li><a href="#2">Prerequisite to use APPIUM</a></li> <li><a href="#22">Install Appium Desktop:</a></li> <li><a href="#3">APPIUM Inspector</a></li> <li><a href="#4">Attach Android Emulator to Appium</a></li> <li><a href="#5">APPIUM Test Case for Native Android App(Calculator)</a></li> <li><a href="#6">Limitations using APPIUM </a></li> <li><a href="#7">Common Encountered Errors and Troubleshooting Steps in Appium</a></li> </ul><h3><a id="1"></a>How APPIUM Works?</h3><ul> <li>Appium is an 'HTTP Server' written using<a href="/node-js-tutorial.html"> a Node.js </a>platform and drives iOS and an Android session using Webdriver JSON wire protocol.
